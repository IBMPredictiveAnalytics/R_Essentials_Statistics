Elements

The elements found in an OXML document correspond to the objects and structures you see in IBM® SPSS® Statistics output. There is an outline, which is a tree structure with an outputTree element as its root, composed of command and textBlock elements.

Commands

A <command> element is the parent node for all the output from one execution of a syntax command. The <command> element has the following attributes:

command. Attribute values are command identifiers. These identifiers are the same as the command identifiers used in the OMS command, and are usually (but not always) the same as the actual command names.

text. The text displayed in the outline pane of the Viewer for the top node of the command's output block. In English, this is usually -- but not -- always -- the same as the command name. This is a localized text string that is affected by the current output language setting.

xml:lang.The output language. Attribute values are standard two-letter language codes (e.g., "en" for English, "es" for Spanish, "fr" for French).

displayOutlineVariables, displayTableVariables. Attribute values are the settings for display of variables and/or variable labels in the outline pane and pivot tables: label, name, or both.

displayOutlineValues, displayTableValues. Attribute values are the settings for display of values and/or value labels in the outline pane and pivot table: label, value, or both.

A <command> element may contain one or more of the following child elements: heading, pivotTable, chart, and textBlock.

Headings

A <heading> element is just a heading in the output outline under a command. It can have any of the sets of common text attributes. In the Viewer, there is a small text block named "Title" under each heading (and command) in the outline that contains the heading text. That text block is not present in the OXML. A <heading> element may contain heading, pivotTable, chart, and textBlock elements.

Text Blocks

Some of the output consists of lines of text. Those are collected in <textBlock> elements. A <textBlock> element may have any number of line elements as children. The body text of each line element is a line of text. The <textBlock> element itself may have any of the sets of common text attributes.

Charts

Each chart is represented by a subdocument described in the Chart XML schema and documented elsewhere.

Pivot Tables

A <pivotTable> element always has at least two attributes:

text. The text displayed in the Viewer outline for the pivot table object. For default table titles, this is a localized text string that is affected by the current output language setting. For table titles that include information about variables used in the table, it is also affected by the current setting for display of variable names and/or variable names.

subType. The table subtype identifier. These identifiers are the same as the subtype identifiers in the OMS command. They are not affected by output language or other user-controlled settings.

A <pivotTable> element can also have a number of common attributes including label, varName, and variable, and two additional attributes:

rowVertical. Attribute values are xs:boolean, indicating whether row labels are displayed vertically rather than horizontally. The attribute value is false by default and not included in OXML. It is only included if the value if true.

columnVertical. Attribute values are xs:boolean, indicating whether column labels are displayed vertically rather than horizontally. The attribute value is false by default and not included in OXML. It is only included if the value if true.
